# SSH流量识别器示例

这是一个完整的从零开始的流量识别示例，演示：

1. 如何读取pcap文件
2. 如何解析网络数据包
3. 如何用DFA状态机匹配特征
4. 如何识别SSH流量

## 编译

```bash
# 安装依赖 (Ubuntu/Debian)
sudo apt-get install libpcap-dev

# 编译
make
```

## 使用方法

```bash
# 识别pcap文件中的SSH流量
./ssh_detector your_capture.pcap
```

## 生成测试数据

```bash
# 方法1: 捕获本地SSH流量
sudo tcpdump -i lo port 22 -c 20 -w test_ssh.pcap
# 然后在另一个终端执行: ssh localhost

# 方法2: 捕获任意网卡的SSH流量
sudo tcpdump -i eth0 port 22 -c 50 -w test_ssh.pcap
```

## 代码结构

```
ssh_detector.c
├── 第一部分: DFA状态机
│   ├── init_dfa()      - 初始化状态转移表
│   └── dfa_match()     - 执行DFA匹配
│
├── 第二部分: 数据包解析
│   └── process_packet() - 解析以太网/IP/TCP并提取payload
│
└── 第三部分: 主函数
    └── main()          - 读取pcap并处理每个包
```

## DFA状态机说明

匹配模式: `"SSH-"`

```
状态转移图:

     'S'       'S'       'H'       '-'
[0] ───→ [1] ───→ [2] ───→ [3] ───→ [[4]]
 ↑        ↑        ↑        ↑        
 └────────┴────────┴────────┴── 其他字符回到[0]

状态说明:
- 状态0: 起始状态
- 状态1: 已匹配 "S"
- 状态2: 已匹配 "SS"  
- 状态3: 已匹配 "SSH"
- 状态4: 已匹配 "SSH-" (接受状态，识别为SSH)
```
